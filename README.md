# AI Interview Coach — Backend (Phase 1)

Phase 1 scope: auth, resume upload/parsing, job description intake, Gemini-based
skill extraction, resume-to-JD match scoring, and personalized question
generation. No live interview loop, voice, or video yet — that's Phases 2-4.

## Setup

```bash
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# edit .env: set DATABASE_URL, JWT_SECRET, GEMINI_API_KEY
# Get a free Gemini key (no credit card needed) at https://aistudio.google.com/apikey
```

You'll need a running PostgreSQL instance. Quickest way locally:

```bash
docker run --name interview-coach-db -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=interview_coach -p 5432:5432 -d postgres:16
```

Run the API:

```bash
uvicorn app.main:app --reload --port 8000
```

Docs at `http://localhost:8000/docs` (Swagger UI, auto-generated by FastAPI).

## API flow (Phase 1)

1. `POST /auth/signup` → `{email, password, full_name}` → creates user
2. `POST /auth/login` → `{email, password}` → returns JWT `access_token`
3. `POST /resumes` (multipart file upload, `Authorization: Bearer <token>`)
   → extracts text (PDF/DOCX) → calls Gemini to extract structured skills
   → returns `Resume` with `extracted_skills`
4. `POST /job-descriptions` → `{title, company, raw_text}` → calls Gemini to
   extract required/nice-to-have skills → returns `JobDescription`
5. `POST /interviews` → `{resume_id, job_description_id, interview_type}`
   → computes resume-to-JD match score (Gemini) → generates 6-8 personalized
   questions (Gemini) → returns `InterviewSession` with `resume_match_score`,
   `skill_match_details`, and the generated `questions` list

## Design notes / what's intentionally deferred

- **Table creation**: uses `Base.metadata.create_all()` for simplicity.
  Switch to Alembic migrations before this touches a shared/prod DB —
  `create_all` can't handle schema changes to existing tables.
- **File storage**: resumes are saved to local disk (`UPLOAD_DIR`). Swap
  `resume.py`'s save step for an S3 upload when you get to the AWS phase.
- **LLM provider**: uses Google's Gemini API (`gemini-2.5-flash` by default) via the
  `google-genai` SDK, with a free tier that needs no credit card — get a key at
  https://aistudio.google.com/apikey. If Google changes free-tier model names/limits
  after this was written, check that page and update `GEMINI_MODEL` in `.env`.
  Free tier has real rate limits (roughly 10 requests/minute as of mid-2026) — if
  you hit 429s while testing quickly, slow down or wait a few seconds between calls.
- **LLM error handling**: if the Gemini call fails during upload (rate limit,
  timeout, malformed JSON), the resume/JD is still saved with
  `extracted_skills` / `extracted_requirements` set to `null` rather than
  failing the whole request. Use `POST /resumes/{id}/reextract` to retry.
- **Scoring fields on `InterviewSession`** (confidence, communication,
  technical, behavioral, hiring_readiness) exist in the schema now but stay
  `null` until Phases 2-5 populate them — avoids another migration later.
- **Auth** is intentionally minimal (email/password + JWT). Add refresh
  tokens / OAuth if needed later.
